# Phase 1: Infrastructure & Scaffolding - Plan

Establish the core development environment and initial scaffolding for the Nexus-OS project.

## Phase Goal

Providing a fully dockerized multi-tier ecosystem (Next.js, Spring Boot, PostgreSQL, Nginx) ready for feature development.

## Implementation Details

### Configuration (Wave 1)

**Task 1: Root Infrastructure & Orchestration**
- **Action:** Create `docker-compose.yml`, `.env.example`, and `nginx.conf` in the project root.
- **Read First:** `.planning/phases/01-infrastructure-scaffold/01-CONTEXT.md`
- **Acceptance Criteria:**
  - `docker-compose.yml` exists and defines services: `frontend`, `backend`, `postgres`, `nginx`.
  - `nginx.conf` exists and routes `/api` to `backend:8080`.
  - `.env.example` exists and includes `DB_HOST`, `DB_USER`, `DB_PASSWORD`.

### Backend Scaffold (Wave 2)

**Task 2: Spring Boot 3 Base**
- **Action:** Create `backend/` directory and `pom.xml` with Spring Boot 3.4.x and Java 21 dependencies.
- **Read First:** `.planning/phases/01-infrastructure-scaffold/01-RESEARCH.md`
- **Acceptance Criteria:**
  - `backend/pom.xml` exists and correctly defines the `spring-boot-starter-parent`.
  - `backend/src/main/resources/application.yml` exists and uses environment variables for DB connection.

**Task 3: Relational Core (Flyway)**
- **Action:** Create the initial migration `V1__Initial_Schema.sql` in `backend/src/main/resources/db/migration/`.
- **Read First:** `NEXUS-Project-Documentation.md` (Table 4.1 to 4.4)
- **Acceptance Criteria:**
  - `V1__Initial_Schema.sql` contains DDL for `users`, `issues`, `docs`, and `certifications` tables.

### Frontend Integration (Wave 3)

**Task 4: App Layout & Modularization**
- **Action:** Update `frontend/src/app/(main)/layout.tsx` to establish the modular navigation and layout architecture.
- **Read First:** `frontend/src/app/layout.tsx`, `.planning/codebase/STRUCTURE.md`
- **Acceptance Criteria:**
  - `app/(main)/layout.tsx` includes a collapsible sidebar and the primary navigation links.

## Verification Criteria

- [ ] `docker-compose up -d` starts all services without exit code 1.
- [ ] Database tables are successfully created via Flyway migrations.
- [ ] Nginx proxy successfully routes requests to the frontend home page.
- [ ] Backend API health check (if implemented) is reachable via `/api/health`.

## must_haves

- Root `docker-compose.yml` for unified orchestration.
- Spring Boot 3 / Java 21 backend scaffold with Maven.
- Flyway migrations for the initial PostgreSQL schema.
- Next.js 16 frontend layout with navigation.
